CitySort

A rules-driven workflow for normalizing and validating location data, with LLM-assisted automations (Anthropic) to speed up triage and rule authoring.

Work with me Repo Live

Problem

Location inputs are noisy: typos, inconsistent formatting, and edge cases (abbreviations, alternate spellings, missing context). The goal of CitySort is to turn messy inputs into clean, consistent, explainable outputs.

Approach

  • Define clear rules and validations so outcomes are consistent and explainable.
  • Build an operator-first UI: fast review, bulk actions, and guardrails for risky changes.
  • Make it observable: audit trails, metrics, and a feedback loop for continuously improving rules.

AI + automation (Anthropic)

CitySort uses Anthropic to assist operators on top of deterministic rules: AI produces suggestions, not silent writes. The goal is to speed up triage while keeping decisions explainable and reversible.

  • Suggest normalized outputs and likely matches
  • Summarise edge cases and highlight missing context
  • Draft rule changes with a before/after preview

Automations for data quality

Beyond AI assistance, CitySort runs background jobs and checks so data stays clean as volume grows.

  • Batch validation to flag anomalies and duplicates
  • Pre-merge checks to prevent rule regressions
  • Audit trails to make changes traceable

Highlights

Human-in-the-loop AI

Anthropic-powered suggestions that speed up operator work, while keeping humans in control of every write.

Automation workflows

Batch jobs and validations that keep the system consistent as the rule set grows.

Traceability + safety

Audit-friendly changes with clarity on what happened, why it happened, and how to undo it.
